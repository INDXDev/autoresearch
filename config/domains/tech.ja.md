# ドメインガイド — テック（日本語）

この実行のトピックを **ソフトウェア・エンジニアリング・プロダクト** のレンズで
調べます: ツール、インフラ、プロトコル、OSS、開発の実践。

## このドメインにおける3層

1. **基礎 — 揺らぎのない事実。** 安定した仕様、広く採用されたツール／プロトコル、
   息の長いアーキテクチャパターン、定番の公式ドキュメント。実務者が依拠できるもの。
2. **最新 — 新しい動き。** リリースやメジャーバージョン更新、RFC、ローンチ、注目の
   新規リポジトリ、障害／ポストモーテムなど。実際に開いたリンクとおおよその日付を添える。
3. **考察 — 作る価値のあるアイデア。** 基礎＋最新を踏まえて *いま* 作れる具体的な
   テックのアイデア: ツール、機能、連携、ライブラリ、アーキテクチャなど試す価値の
   あるもの。明確な提案としてのアイデア、その背景にある着眼点や埋めるべき隙間（なぜ今）、
   検証する最初の試作／スパイク、主なリスクを書く。採用判断（「用途Yなら Xは本番投入可能」）
   もここに含めてよい。

## 優先する情報源

まず `config/priority_sources.md` を WebFetch。次に公式ドキュメントと変更履歴、
GitHub のリリース／Issue、標準化団体（IETF RFC, W3C, WHATWG）、信頼できる
エンジニアリングブログ、シグナルとしての Hacker News（HN リンクの一次情報を必ず確認）を優先。

## 良い項目とは

作り手にとって意味のある具体性: バージョン、ベンチマーク、破壊的変更、実際の
トレードオフ。集約された感想より一次情報（リリースノート、リポジトリ、仕様）を優先。

## 必ず守ること

- ラボの焦点と制約のため `config/research_topics.md` を読む。
- リリース・バージョン・リポジトリ・URL を捏造しない — 見つけたものだけを含める。
- すべてのテキスト項目を **日本語** で書く。
